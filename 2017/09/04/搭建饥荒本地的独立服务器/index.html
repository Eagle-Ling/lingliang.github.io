搭建本地的饥荒独立服务器

发表于 2017-09-04 | 分类于 其他 | | 阅读次数

在本地搭建独立的饥荒服务器（Windows系统）

可能好多人和好友一起玩饥荒时都会遇到这样的问题：

  1. 主机好友玩着玩着突然有事离线，然后自己玩得正嗨，突然就与主机连接中断了，很沮丧。
  2. 自己作为主机时不想玩了，然后好友还意犹未尽，但也不得不和你一起下线。
  3. 世界时间达到几百天以后越玩越卡顿，好友也表示延迟越来越高了，游戏体验越来越差。

如果你在玩饥荒时遇到如上情况的困扰，那么你可以在自己的电脑上搭建一个饥荒的本地服务器，当你突然有事要离开一会儿，即使退出了游戏，只要你的电脑在后台运行着服务器，那么你的好友依然可以在你创建的世界快乐的玩耍！而且玩到后期也不会因为世界里的素材太多而变得卡顿！

事不宜迟，先说说需要准备的材料：

  • Steam正版饥荒 -_-

第一步：创建一个世界

创建一个世界，选择好你想用的Mod，关于mod我建议能少尽量少，能大大提高游戏的稳定性，而且添加的mod最好不是常常更新的那种。

在这里我选择了第三个存档创建了一个世界。

第二步：配置token

  1. 进入游戏菜单界面，点击右下角的个人资料

  2. 然后会出现下面的页面，点击那个Generate Server Token按钮。

然后会在token列表的最上面生成一个最新的token,复制这个token，下一步要用。（PS：这个token是Klei防盗版的一种的限制，没有这个token就不能创建世界，所以盗版玩家就不要浪费时间在这篇帖子上了-_-||)

3.上面我已经创建了一个世界了，然后就能找目录<文档/Klei/DoNotStarveTogether/Cluster_3>，其中Cluster_数字对应你创建的第几个存档,比如我创建的是第三个存档，就是Cluster3)，如图：

点进去这个目录下能看到下面几个文件：Master(主世界数据)，Caves(洞穴实际数据，如果选择添加洞穴的话)，cluster.ini(世界的配置文件)。然后需要在这个目录中添加一个文件叫cluster_token.txt，把之前复制的token粘贴进去，保存。

第四步“创建脚本”

先不要激动!接下来是最后一步。
进入steam的安装目录，进入steamapps/common/Don’t Starve Together/bin，可以看到一个dontstarve_dedicated_server_nullrenderer.exe这个文件，从这个应用程序的名字就可以知道，就需要通过它来创建一个无画面的服务器。
  创建一个文件start.bat,在里面写入以下代码：

dontstarve_dedicated_server_nullrenderer -console -cluster Cluster_3 -shard Master

注意了，这里Cluster_3表示你创建的是第三个存档的世界。然后创建一个文件start_cave.bat,在里面写入：

dontstarve_dedicated_server_nullrenderer -console -cluster Cluster_3 -shard Caves


然后启动上面两个刚创建的脚本，等它启动完成

好了，你可以在浏览游戏中搜索创建的世界名字，比如我的是：


细节下面有一个服务器图标，说明你已经完成了饥荒独立服务器的搭建，以后玩游戏之前先启动上面两个脚本（相当于启动本地服务器），再搜索出世界，就能愉快的和好友玩耍了！当然，本机的话需要到LAN中才能找到世界。玩到后期也不会卡。

祝您游戏愉快~
